Using method editor
There are three GUIs using the method editor. One is defining methods in category definition window (Refer to "Creating a new method" for more details). Another is creating behaviors in behavior network window (Refer to "Creating a new behavior" for more details). The third one is creating general dynamics (Refer to "Creating general dynamics" for more details).

The interface of method editor is shown below.


You can type the character '@' to see all available system APIs, and self-defined methods or fields. The methods and fields are shown in a popup list as below. The methods and fields are categorized and sorted. Both keyboard and mouse can be used to select an item.


In the case of invalid method definition, the errors of syntax or semantics will be shown in the popup window. A sample syntax error is shown below.


The error can be reported to BehaviorSim development group by clicking "Report Bugs" button. Type in your email address and you will get reply once the reported bugs have been solved. No reply will provide if the email address is left blank.
