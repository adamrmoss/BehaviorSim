About BehaviorSim
BehaivorSim is a learning environment for behavior-based agent. It supports learning of behavior-based control by defining simulated agents in an intuitive manner corresponding to the behavior-based paradigm. It allows educators and students to develop and exercise their own behavior-based applications by setting up entities, behaviors and other primitives without involving significant programming effort. The term behavior-based agent is adapted from behavior-based robotics. Here an agent is a general term that can be a robot, an animat, an artificial agent in AI, a character in game design, or a simulated individual in social crowd simulation.

BehaviorSim is an ongoing project led by Dr. Xiaolin Hu in the computer science department at GSU. Early development of BehaviorSim was supported by a seed grant from the GSU Brain and Behavior program, in collaboration with Prof. Donald Edwards.

Introduction to BehaviorSim environment
The BehaviorSim environment is a JAVA application with simple and user-friendly interface. The following figure shows a snapshot of the environment. Besides menus and toolbars, the environment contains several important panels. Topmost panel shows application name and the full path. The middle left panel shows the navigation tree with application configurations such as the virtual simulation world, categories, and all entities (stationary entity or autonomous agent) & their specific field information. While the middle right panel shows the application editor where an entity can be easily created by simply dragging a category object into the virtual world (blue area). The bottom panel contains three regions which show version information, messages of every operation such as creating a category, and application status (i.e. created, loaded, not saved, saved, etc.) respectively.
